#  LLM Actor-Critic 长文本推理工作流

本项目实现了一个基于 **LangGraph** 框架的“行动者-评判者”（Actor-Critic）高阶工程化架构。它专为解决大语言模型（LLM）在长文本推理、复杂代码生成及深度逻辑分析中的“幻觉”与“逻辑断层”问题而设计。

通过引入 **LLM-as-a-Judge（大模型作为评判者）** 范式，本系统构建了一个具备“反思与修正”能力的闭环工作流，模拟人类专家“起草-审阅-修订”的认知过程。

##  核心架构理念

在现代 LLM 工程化应用中，传统的单次 Prompt 调用往往难以应对复杂的长文本任务。本架构通过以下三个核心组件实现质量跃升：

* **行动者（Actor）**：负责执行核心任务，生成复杂的推理链、代码或长文本草稿。
* **评判者（Critic）**：作为独立的“质检员”，对 Actor 的产出进行步骤级或整体级的质量评估。它不输出简单的标量分数，而是提供具有高度可操作性的**自然语言反馈**。
* **闭环（Loop）**：基于 LangGraph 的状态化图结构，Actor 根据 Critic 的反馈进行定向修正，直到产出符合预设的高质量标准。

##  工程化价值解析

这段代码不仅仅是一个简单的循环调用，它体现了目前构建高可靠性 AI Agent 的主流工程范式：

1. **自然语言作为反馈信号（Natural Language Critique）**
传统的强化学习 Actor-Critic 架构中，Critic 通常输出一个标量值（如 0.8 分）。而在长文本推理中，单纯的分数无法指导模型如何修改。本架构中，Critic 输出的是**自然语言建议**（例如：“你的快速排序没有处理重复元素，且缺少时间复杂度解释”）。这种反馈对 LLM 来说比数字更具可操作性，能精准指导 Actor 进行定向修正。

2. **步骤级的信用分配（Step-level Credit Assignment）**
长文本推理最怕“秋后算账”（即生成完几千字后才发现开头逻辑错了）。在这个架构中，Critic 在 Actor 生成完一个完整草稿后立即介入。它将最终的质量要求，拆解成了每一步具体的“修改意见”，完美解决了长文本生成中的信用分配难题，防止错误在长链路中累积。

3. **有状态的循环工作流（Stateful Loop）**
通过 `LangGraph` 的 `StateGraph`，我们维护了 `draft`（草稿）、`critique`（反馈）和 `revision_count`（修正次数）等状态。这模拟了人类专家“写作-审阅-修改”的真实认知过程，比单次调用的 Prompt 工程具备极强的容错性和逻辑深度。

##  适用场景

* **复杂代码生成**：Actor 编写代码，Critic 充当代码审查员（Code Reviewer），检查 Bug、边界条件及代码规范。
* **深度研报/文章撰写**：Actor 负责起草，Critic 检查逻辑连贯性、数据准确性及论证深度。
* **数学与逻辑推理**：Actor 给出解题步骤，Critic 逐步验证推导过程的正确性。
